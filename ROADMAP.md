# WebThief 高级交互还原路线图 (Roadmap to 1:1 Fidelity)

目前的 v2 版本已解决视觉保真度问题。下一步，WebThief 将挑战克隆界的“圣杯”：**动态交互的一比一还原**。

## 🎯 核心目标 1：官方实时二维码 (Live QR Code)
目前市场上 99% 的克隆工具只能捕获静态二维码。我们的目标是让克隆页面的二维码具备“生命”。

- **技术挑战：脚本生命周期恢复 (Script Lifecycle Recovery)**
    - 研究如何精准识别并保留 Steam 核心 Auth 脚本及其依赖逻辑，使其在离线环境下仍能建立官方握手。
    - 探索模拟官方通信环境的各种手段，确保二维码能实时拉取、自动刷新并监听扫码反馈。
    - **最终效果**：克隆登录页上的二维码会随官方节奏自动刷新，且扫码交互与官方原版完全一致。

## 🎯 核心目标 2：完全体交互菜单 (Expandable Mega-Menus)
由于 Steam 使用 React 进行动态渲染，传统的静态 HTML 很难保留复杂的下拉交互。

- **技术突破：React 销毁拦截 (Unmount Interception)**
    - 注入底层补丁，在克隆采集期间劫持 React 的组件卸载函数。
    - 强制所有触发过的“分类”、“商店”下拉层驻留在 DOM 树中，即使鼠标离开也不被销毁。
- **技术突破：交互语义恢复 (UI Semantic Thawing)**
    - 自动分析 React 组件的样式的触发逻辑，将 JS 驱动的显隐状态转换为更强健的 CSS 处理。
    - **最终效果**：不管是“类别”的大型格栅菜单，还是“商店”的悬停浮窗，在克隆页中都能点击或悬停正常展开，逻辑一比一复刻。

## 🚀 未来展望
- **交互自适应修复**：利用小模型自动修复克隆后失效的 JS 事件监听器。
- **全自动注入层**：一键注入自定义的登录捕获（Phishing Simulation）逻辑，与一比一还原的菜单无缝结合。
